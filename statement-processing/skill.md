## Environment

Secrets are managed via **Doppler** (auto-configured in repo directory). Use the venv Python and prefix with `doppler run --`:

```bash
cd /Users/troybrave/.claude/.CLI/statement-processing
doppler run -- .venv/bin/python3 statement-import/test_ocr.py --file ~/Downloads/statement.pdf
doppler run -- .venv/bin/python3 statement-import/import_pipeline.py --client jose-sandoval
```

**Always use `.venv/bin/python3`** — system Python does not have the required dependencies.

| Secret | Purpose |
|--------|---------|
| `GEMINI_API_KEY` | Gemini 3 Flash OCR |
| `AIRTABLE_PERSONAL_ACCESS_TOKEN` | Airtable API access |

## Architecture

```
PDF Statement
     |
     v
[Gemini 3 Flash]        statement-import/test_ocr.py
  Agentic Vision OCR     - Any bank format, no parser code
  Code execution          - Self-verifies arithmetic
  ~$0.006/statement       - Returns structured JSON
     |
     v
Structured JSON
     |
     v
[Claude]                 statement-import/import_pipeline.py
  Orchestration           - Client resolution + Airtable record creation
  Dedup                   - Fuzzy match vs Plaid (+-3 days)
  Enrichment              - Merchant name cleanup
     |
     v
Airtable Records
```

## Amount Sign Convention

- **NEGATIVE** = money IN (deposits, credits, refunds)
- **POSITIVE** = money OUT (withdrawals, debits, fees, purchases)

## Client Folder Convention

All client work lives in `./clients/<kebab-case-name>/`:
- `config.json` -- Airtable base ID, account mappings
- `import_config.json` -- Auto-generated by setup-import
- `statements/raw/` -- Renamed originals (archive, never touched)
- `statements/pending/` -- Ready for import
- `statements/processed/` -- Auto-moved after successful import
- `notes.md` -- Running log of imports
